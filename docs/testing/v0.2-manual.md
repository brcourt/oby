# oby v0.2 — manual functional test prompts

Append to the v0.1 manual smoke suite. v0.1 scenarios from
`docs/testing/v0.1-manual.md` continue to apply.

## How to use

1. `oby claude` in a wrapped terminal.
2. Paste a scenario's **Prompt** block.
3. Press **Ctrl-G** to view the feed.
4. Compare against the **Expected** block.

## V0.2-1. `| grep` pre-filter capture

**Prompt:**

```
Run: cat /etc/hosts | grep 127
```

**Expected:**

- `✓ Bash  cat /etc/hosts | grep 127` on `main`.
- `[stdout-pre-grep]` chunk containing 3 lines before + match + 3 lines after each `127` line (≈7 lines per match), NOT the entire `/etc/hosts`. (Window size configurable via `OBS_GREP_BEFORE_LINES` / `OBS_GREP_AFTER_LINES`.)
- `[stdout]` chunk containing only lines matching `127` (what claude sees).

**Verifies:** `inject_pre_filter_tees` injects a tee on the producer side; the matched chunk goes to stdout, the upstream goes to stdout-pre-grep.

## V0.2-2. `| head` window capture

**Prompt:**

```
Run: cat /etc/passwd | head -n 3
```

**Expected:**

- `✓ Bash  cat /etc/passwd | head -n 3`.
- `[stdout-pre-head]` chunk: lines 4–6 of `/etc/passwd` (the 3 lines immediately AFTER head's truncation boundary). NOT the entire file. (Peek count configurable via `OBS_HEAD_PEEK_LINES`.)
- `[stdout]` chunk: first 3 lines only.

## V0.2-3. `| tail` window capture

**Prompt:**

```
Run: cat /etc/passwd | tail -n 3
```

**Expected:**

- `[stdout-pre-tail]` chunk: the 3 lines immediately BEFORE tail's boundary (the last 3 non-tail lines). NOT the entire file. (Peek count configurable via `OBS_TAIL_PEEK_LINES`.)
- `[stdout]` chunk: last 3 lines only.

## V0.2-4. `> FILE` redirect capture

**Prompt:**

```
Run: echo "hello from oby v0.2" > /tmp/oby-v0.2-test.txt
```

**Expected:**

- `✓ Bash  echo "hello from oby v0.2" > /tmp/oby-v0.2-test.txt`.
- `[stdout-to-file]` chunk: `hello from oby v0.2`.
- `[stdout]` chunk: empty (claude saw nothing; everything went to the file).
- The file `/tmp/oby-v0.2-test.txt` actually exists with that content.

## V0.2-5. `>> FILE` append capture

**Prompt:**

```
Run: echo "appended" >> /tmp/oby-v0.2-test.txt
```

**Expected:**

- `[stdout-appended-file]` chunk: `appended`.
- The file now ends with two lines (`hello from oby v0.2` then `appended`).

## V0.2-6. Quoted `|` and `>` NOT rewritten

**Prompt:**

```
Run: echo "this | grep is not piped and > is just text"
```

**Expected:**

- One `[stdout]` chunk with the literal text.
- NO `[stdout-pre-grep]` or `[stdout-to-file]` chunks (scanner correctly excluded the quoted metachars).

## V0.2-7. xtrace under single command

**Prompt:**

```
Run: echo trace-test
```

**Expected:**

- `[stdout]` chunk: `trace-test`.
- `[xtrace]` chunk: `+ echo trace-test` (one line, exact).

**Note:** under zsh (default macOS shell), the `[xtrace]` chunk is in zsh's native format with the leading `+(SOURCE):LINE> ` prefix, e.g.:

```
+(eval):3> echo trace-test
```

instead of bash's `+ echo trace-test`. v0.2.1 preserves zsh's richer trace format intentionally (more info about where the trace originated).

## V0.2-8. xtrace under multi-statement script

**Prompt:**

```
Run: for i in 1 2 3; do echo "iter $i"; done
```

**Expected:**

- `[stdout]` chunk: `iter 1\niter 2\niter 3`.
- `[xtrace]` chunk: lines like
  ```
  + for i in 1 2 3
  + echo 'iter 1'
  + echo 'iter 2'
  + echo 'iter 3'
  ```
  (Exact prefix `+ ` per command. Bash may or may not show the `for` line depending on version.)

**Note:** under zsh, the chunk lines use the zsh-native prefix:

```
+(eval):3> echo iter 1
+(eval):3> echo iter 2
+(eval):3> echo iter 3
```

zsh tends to inline the loop variable expansion at each iteration, so the trace shows the expanded form per command.

## V0.2-11. `grep -v` skips pre-filter capture

**Prompt:**

```
Run: cat /etc/hosts | grep -v '^#'
```

**Expected:**

- `✓ Bash  cat /etc/hosts | grep -v '^#'`.
- `[stdout]` chunk: only non-comment lines.
- **NO `[stdout-pre-grep]` chunk.** v0.2.2 skips pre-filter capture for `grep -v` because showing the excluded lines (comments here) is generally noise.

**Verifies:** the `grep_is_invert_match` detection in `pre_filter.rs` triggers the skip path.

## V0.2-12. Env-var override

**Pre-step:** quit `oby claude`, then in your shell:

```bash
export OBS_GREP_BEFORE_LINES=1
export OBS_GREP_AFTER_LINES=1
oby claude
```

**Prompt:**

```
Run: cat /etc/hosts | grep 127
```

**Expected:**

- `[stdout-pre-grep]` chunk: 1 line before + match + 1 line after (≈3 lines per match), not the default 7.

**Verifies:** `PreFilterConfig::from_env()` reads the env vars and the values are embedded in the rewrite.

## V0.2-9. Nesting guard

In a wrapped session, paste:

```
Run: oby claude
```

**Expected:**

- The bash command exits non-zero.
- Feed entry shows `[stderr]` chunk:
  ```
  oby: refusing to nest — OBS_ACTIVE is already set.
       you're trying to start an oby session from inside an existing one.
       …
  ```
- The outer session continues normally (no freeze).

## V0.2-10. `oby-hook --version`

Outside any wrapped session:

```bash
oby-hook --version
```

**Expected:** prints `oby-hook 0.2.2` to stdout, exits 0. (Adjust if running against an earlier release; this scenario verifies the binary version matches the installed package.)
